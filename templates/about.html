Project Information & Technical Specifications

[Project Overview]

Banks and lending institutions receive millions of credit card applications annually. Manually reviewing and scoring each application is slow, operationally expensive, inconsistent, and highly vulnerable to human bias or error.

This project builds a state-of-the-art automated decisioning engine using historical customer profile credit card payment records. The machine learning pipeline trains multiple classifiers and selects the optimal model to render instant approval determinations while categorizing risk classes under high validation standards.

[Data Preprocessing Pipeline]

  • Merging & Target Formulation: Application profiles (`application_record.csv`) are matched with credit monthly payment statements (`credit_record.csv`). Applicants are labeled as "Good" (Approved) or "Bad" (Rejected) depending on whether they ever cross a 60-day delinquency threshold.
  • Handling Missing Data: Undefined jobs in the `OCCUPATION_TYPE` column are resolved based on days of employment (e.g. mapping retired/unemployed rows to "Pensioner" and others to "Other/Unspecified").
  • Feature Scaling: Numeric attributes like income, age, family counts, and job tenure are standardized using `StandardScaler`.
  • Categorical Encoding: Multi-class values are converted using a robust `OneHotEncoder` with ignore options for unseen parameters in incoming online queries.
[IBM Watson Architecture]

This project is designed for enterprise scaling with:

  1. Models serialized to pickled files (`scaler.pkl`, `encoder.pkl`, `best_model.pkl`).
  2. Serialized structures pushed to IBM Cloud Object Storage (COS).
  3. Online deployment via the **IBM Watson Machine Learning (WML)** service registry.
  4. Online scoring endpoints accessed through Flask REST calls.

Machine Learning Models Comparison

Logistic Regression

A classic linear model serving as our baseline. It models the probability of credit approval using the logistic sigmoid function. Highly interpretable but lacks capability for capturing complex non-linear feature interactions.

Decision Tree Classifier

Creates decision boundaries by splitting data based on information gain or Gini impurity thresholds. Capable of modeling complex conditions, but prone to high variance and overfitting if uncontrolled.

Random Forest Classifier

An ensemble bagging method that constructs numerous independent decision trees at training time and outputs the mode of classes or average prediction. Highly robust against overfitting and offers excellent feature importance insights.

XGBoost Classifier

A state-of-the-art gradient boosted decision tree algorithm. It iteratively builds trees to correct errors made by previous estimators. Outperforms other algorithms on tabular data with regularization controls and parallel execution.
